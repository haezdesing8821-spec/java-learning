# Tutorial De Java - Session 7. / Los Operadores

## 07.8 Operadores Ternario en java

### ¿Qué es?

En Java, el operador ternario (`?:`s) es una forma abreviada de escribir una sentencia `if`-`else` en una sola línea.
Su sintaxis es:

* **`condición`** → expresión booleana que se evalúa (`true` o `false`).

* **`valorSiVerdadero`** → valor que se asigna si la condición es `verdadera`.

* **`valorSiFalso`** → valor que se asigna si la condición es `falsa`.

* El **`operador ternario`** es un operador condicional que evalúa una expresión booleana y retorna uno de dos valores posibles según el resultado.

| Operador | Descripción          | Sintaxis                                 | Resultado                      |
| -------- | -------------------- | ---------------------------------------- | ------------------------------ |
| `?:`     | Operador condicional | `condición ? valorSiTrue : valorSiFalse` | Retorna uno de los dos valores |

### EJEMPLO DE USO

**VALÓRES INICIALES**

```java
int a = 5;
int b = 8;
```

**USO DEL OPERADOR TERNARIO CON NÚMEROS**

```java
int mayor = (a > b) ? a : b;
```

**EVALUACIÓN PASO A PASO**

Se evalúa la condición: `a > b` →  `5 > 8`→ `false`

Como es `false`, se vuelve `b` 

Resultado final: `mayor = 8`

**EL CÓDIGO COMPLETO SE VERÍA ASÍ:**

```java
public class TernaryExample {
    public static void main(String[] args) {

        int a = 5;
        int b = 8;

        int mayor = (a > b) ? a : b;

        System.out.println("El mayor es: " + mayor); // 8
    }
}
```

**USO DEL OPERADOR TERNARIO CON BOOLEANOS**

```java
boolean esMayor = (a > b) ? true : false;
System.out.println(esMayor); // false
```

**Nota técnica importante**

El operador ternario no reemplaza a `if`-`else`, pero es útil para:

* Asignaciones simples

* Expresiones compactas

Ambos valores ( `valorSiTrue` y `valorSiFalse`) deben ser compatibles en tipo.

### Comparación rápida

| Estructura | Uso                                      |
| ---------- | ---------------------------------------- |
| `if-else`  | Lógica compleja, múltiples instrucciones |
| `?:`       | Decisiones simples en una sola expresión |

### Ventajas y buenas prácticas

✅ Código más compacto que un if-else tradicional.

✅ Útil para asignaciones rápidas.

**No abusar:** si la lógica es compleja, es mejor usar `if`-`else` para mantener la legibilidad.
